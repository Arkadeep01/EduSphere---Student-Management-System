# Admin Backend Mapping

| Admin Page | Required Models | Required APIs | Required Services | Required Permissions | Required Relationships |
|---|---|---|---|---|---|
| **Dashboard** (`/admin/dashboard`) | - `StudentProfile` (student count) <br> - `TeacherProfile` (teacher count) <br> - `Class` (class count) <br> - `Fee` (revenue, pending fees) <br> - `Attendance` (student attendance %) <br> - `FacultyAttendance` (faculty analytics) <br> - `Exam` (upcoming exams) <br> - `Event` (upcoming events) <br> - `Notification` (admin notifications) <br> - `Subject` (exam performance) | - `GET /admin/dashboard/summary/` – returns overall statistics (students, teachers, classes, revenue, fees, attendance, exams, events) <br> - `GET /admin/dashboard/student-growth/` – time‑series data for student enrolment <br> - `GET /admin/dashboard/attendance-data/` – weekly attendance percentages <br> - `GET /admin/dashboard/exam-performance/` – average marks per subject <br> - `GET /admin/dashboard/events/` – list of upcoming events | - **DashboardStatsService** – aggregates data from multiple models <br> - **ReportingService** – builds chart datasets <br> - **NotificationService** – fetches recent admin notifications | `IsAdmin` (custom permission ensuring only admin users may call) | - `StudentProfile` → `User` (one‑to‑one) <br> - `TeacherProfile` → `User` (one‑to‑one) <br> - `Attendance` → `StudentProfile` (many‑to‑one) <br> - `FacultyAttendance` → `TeacherProfile` (many‑to‑one) <br> - `Exam` → `Subject` (many‑to‑one) <br> - `Event` independent <br> - `Notification` → `User` (many‑to‑one) |
| **Students** (`/admin/students`) | - `StudentProfile` <br> - `Class` (new) <br> - `Section` (new, part of Class) <br> - `StudentSubject` (subject allocations & status) <br> - `SubjectRequestControl` (global toggle) <br> - `Fee` (per‑student fee records) <br> - `DocumentStorage` (student documents) <br> - `ExportLog` (record of data exports) <br> - `Notification` (admin → student) | - `GET /admin/classes/` – list classes with summary stats <br> - `GET /admin/classes/{class_id}/` – class detail (sections, subjects, teacher allocation) <br> - `GET /admin/classes/{class_id}/students/` – paginated students in a class <br> - `GET /admin/students/{student_id}/` – student detail (personal, academic, attendance, documents, notifications) <br> - `POST /admin/students/` – create new student <br> - `PATCH /admin/students/{id}/` – update student data <br> - `GET /admin/fees/{class_id}/` – fee report per class (paid/unpaid) <br> - `POST /admin/exports/student/` – trigger CSV/Excel/PDF export; logs entry in `ExportLog` <br> - `GET /admin/subject-request/` – retrieve current toggle state <br> - `PATCH /admin/subject-request/` – enable/disable subject requests <br> - `POST /admin/notifications/student/{id}/` – send notification to a student | - **StudentManagementService** – CRUD + enrolment logic <br> - **ClassManagementService** – class overview & detail <br> - **FeeManagementService** – compute fee status, generate reports <br> - **ExportService** – build CSV/Excel/PDF and create `ExportLog` <br> - **SubjectRequestService** – read/write global request toggle <br> - **NotificationService** – create admin‑sent notifications | `IsAdmin` | - `StudentProfile` → `User` (one‑to‑one) <br> - `StudentProfile` → `Class` (many‑to‑one) <br> - `StudentSubject` → `StudentProfile` & `Subject` (many‑to‑one) <br> - `Fee` → `StudentProfile` (many‑to‑one) <br> - `DocumentStorage` → `StudentProfile` (many‑to‑one) <br> - `ExportLog` records which admin user performed export |
| **Teachers** (`/admin/teachers`) | - `TeacherProfile` <br> - `Subject` (subject list) <br> - `TeacherClassAssignment` (class‑teacher mapping) <br> - `TeacherSubjectAllocation` (new) <br> - `Notification` (admin → teacher) <br> - `DocumentStorage` (teacher documents) | - `GET /admin/teachers/` – list all teachers with summary info <br> - `POST /admin/teachers/` – create new teacher (auto‑creates profile) <br> - `PATCH /admin/teachers/{id}/` – update teacher details <br> - `POST /admin/teachers/{id}/notify/` – send a notification <br> - `POST /admin/teachers/{id}/assign-class/` – link teacher to a class (creates `TeacherClassAssignment`) <br> - `POST /admin/teachers/{id}/allocate-subject/` – create `TeacherSubjectAllocation` record | - **TeacherManagementService** – CRUD, profile auto‑creation <br> - **AllocationService** – handles class‑teacher and subject‑teacher assignments <br> - **NotificationService** – push messages to teachers <br> - **DocumentService** – manage teacher uploads | `IsAdmin` | - `TeacherProfile` → `User` (one‑to‑one) <br> - `TeacherClassAssignment` → `TeacherProfile` & `Class` (many‑to‑one) <br> - `TeacherSubjectAllocation` → `TeacherProfile`, `Subject`, and optionally `Class` (many‑to‑one) <br> - `Notification` → `User` (many‑to‑one) |
| **Classes** (`/admin/classes`) | - `Class` (new) <br> - `Section` (embedded in `Class`) <br> - `Subject` <br> - `ClassTeacherAssignment` (new) <br> - `TeacherClassAssignment` (existing) <br> - `TeacherSubjectAllocation` (new) <br> - `TimetableEntry` (existing) | - `GET /admin/classes/` – list class cards <br> - `POST /admin/classes/` – create a new class with sections and capacity <br> - `GET /admin/classes/{id}/` – detailed view (sections, subjects, teachers, timetable) <br> - `PATCH /admin/classes/{id}/` – update class info <br> - `POST /admin/classes/{id}/assign-teacher/` – create `ClassTeacherAssignment` <br> - `POST /admin/classes/{id}/assign-subject/` – create `TeacherSubjectAllocation` for a subject in this class <br> - `POST /admin/classes/{id}/timetable/` – bulk create/update `TimetableEntry` records | - **ClassManagementService** – CRUD, capacity logic, section handling <br> - **TeacherAssignmentService** – manage `ClassTeacherAssignment` and `TeacherClassAssignment` <br> - **SubjectAllocationService** – manage `TeacherSubjectAllocation` <br> - **TimetableService** – create, edit, validate timetable entries | `IsAdmin` | - `Class` → many `Section`s (one‑to‑many) <br> - `Class` → many `StudentProfile`s (one‑to‑many) <br> - `ClassTeacherAssignment` → `Class` & `TeacherProfile` (many‑to‑one) <br> - `TeacherSubjectAllocation` → `Class`, `Subject`, `TeacherProfile` (many‑to‑one) <br> - `TimetableEntry` → `Class` (via `class_name`) and `TeacherProfile` <br> - `Subject` ↔ `Class` through allocation records |
| **Attendance** (`/admin/attendance`) | - `Attendance` (student daily records) <br> - `FacultyAttendance` (new) | - `GET /admin/attendance/analytics/` – returns school‑wide, class‑wise, weekly trends <br> - `GET /admin/faculty-attendance/` – list of faculty with today’s status <br> - `PATCH /admin/faculty-attendance/{teacher_id}/` – update present/absent/leave/half‑day for today | - **AttendanceAnalyticsService** – compute aggregates, weekly trends <br> - **FacultyAttendanceService** – mark and retrieve faculty attendance status | `IsAdmin` | - `Attendance` → `StudentProfile` (many‑to‑one) <br> - `FacultyAttendance` → `TeacherProfile` (many‑to‑one) |
| **Exams** (`/admin/exams`) | - `Exam` (new) <br> - `ExamSchedule` (new) <br> - `AnswerScriptUpload` (new) <br> - `EvaluationTracking` (new) <br> - `PublishedResult` (new) <br> - `Subject` <br> - `StudentProfile` (for answer scripts) <br> - `TeacherProfile` (evaluator) | - `GET /admin/exams/` – list exams with status <br> - `POST /admin/exams/` – create exam (draft) <br> - `PATCH /admin/exams/{id}/publish/` – change status to *published* <br> - `PATCH /admin/exams/{id}/archive/` – archive exam <br> - `POST /admin/exams/{id}/schedule/` – create/update `ExamSchedule` <br> - `POST /admin/exams/{id}/answer-scripts/` – upload scripts (creates `AnswerScriptUpload`) <br> - `GET /admin/evaluation-tracking/` – list evaluation progress per teacher <br> - `POST /admin/published-results/` – publish final results (creates `PublishedResult`) <br> - `GET /admin/results/` – view published results | - **ExamService** – CRUD, status transitions, validation <br> - **SchedulingService** – generate & edit `ExamSchedule` <br> - **AnswerScriptService** – handle file upload, link to student & teacher <br> - **EvaluationService** – track progress, update `EvaluationTracking` <br> - **ResultPublicationService** – compute, store, and expose `PublishedResult` | `IsAdmin` | - `Exam` → many `ExamSchedule`s (one‑to‑many) <br> - `AnswerScriptUpload` → `Exam`, `StudentProfile`, `TeacherProfile` (many‑to‑one) <br> - `EvaluationTracking` → `AnswerScriptUpload` & `TeacherProfile` (many‑to‑one) <br> - `PublishedResult` → `Exam` & `StudentProfile` (many‑to‑one) |
| **Events** (`/admin/events`) | - `Event` (new) | - `GET /admin/events/` – list all events <br> - `POST /admin/events/` – create event (draft) <br> - `PATCH /admin/events/{id}/publish/` – set status *published* <br> - `PATCH /admin/events/{id}/archive/` – archive event <br> - `DELETE /admin/events/{id}/` – remove event | - **EventService** – CRUD, status management, banner image handling (through `DocumentStorage`) | `IsAdmin` | - `Event` independent; may reference media stored in `DocumentStorage` |
| **Contacts** (`/admin/contacts`) | - `ContactSubmission` (new) <br> - `Notification` (admin → contact via email) | - `GET /admin/contacts/` – list submissions with filters <br> - `PATCH /admin/contacts/{id}/status/` – change to *read*, *resolved*, *pending* <br> - `DELETE /admin/contacts/{id}/` – delete record <br> - `POST /admin/contacts/{id}/email/` – send reply email (records action) | - **ContactManagementService** – list, filter, status updates <br> - **EmailService** – send email replies (uses template) | `IsAdmin` | - `ContactSubmission` independent; may store reference to user if email matches a registered user |
| **Admissions** (`/admin/admissions`) | - `AdmissionApplication` (new) <br> - `AdmissionVerification` (new) <br> - `AdmissionDocument` (could reuse `AdmissionDocument` from student app) <br> - `StudentRegistrationLog` (new) <br> - `DocumentStorage` (uploaded docs) <br> - `NotificationBroadcast` (new, for email notifications) <br> - `AdmissionPageContent` (CMS) | - `GET /admin/admissions/` – list applications <br> - `GET /admin/admissions/{id}/` – detail view with documents & marks <br> - `PATCH /admin/admissions/{id}/approve/` – set status *approved* <br> - `PATCH /admin/admissions/{id}/reject/` – set status *rejected* <br> - `POST /admin/admissions/{id}/request-info/` – create a verification task <br> - `POST /admin/admissions/{id}/create-student/` – trigger student account creation (logs in `StudentRegistrationLog`) <br> - `POST /admin/admissions/{id}/documents/` – upload additional docs <br> - `GET /admin/admissions/stats/` – dashboard metrics (total, appeared, passed, failed, pending verification, selected, rejected) | - **AdmissionService** – CRUD, status workflow, verification handling <br> - **DocumentService** – store/retrieve admission docs (uses `DocumentStorage`) <br> - **StudentAccountService** – create `StudentProfile` + credentials after approval, log in `StudentRegistrationLog` <br> - **NotificationService** – send email updates to applicants <br> - **AnalyticsService** – aggregate admission statistics | `IsAdmin` | - `AdmissionApplication` → many `AdmissionDocument`s <br> - `AdmissionVerification` → `AdmissionApplication` (one‑to‑one) <br> - `StudentRegistrationLog` → `StudentProfile` (one‑to‑one) <br> - `NotificationBroadcast` → applicant email address |
| **Settings** (`/admin/settings`) | - `AboutPageContent` (new) <br> - `GalleryImage` (new) <br> - `HomepageFeaturedImage` (new) <br> - `AdmissionPageContent` (new) <br> - `DocumentStorage` (uploaded media) | - `GET /admin/settings/about/` – retrieve/about data <br> - `PATCH /admin/settings/about/` – update text, video URL, featured video title, top students <br> - `GET /admin/settings/gallery/` – list gallery images <br> - `POST /admin/settings/gallery/` – upload new image <br> - `PATCH /admin/settings/gallery/{id}/` – reorder / set featured / delete <br> - `GET /admin/settings/home/` – retrieve homepage featured images <br> - `POST /admin/settings/home/` – upload / reorder / delete images <br> - `GET /admin/settings/admission/` – retrieve admission page data (fees, dates, notices) <br> - `PATCH /admin/settings/admission/` – update banner, fees, capacity, dates, notices <br> - (similar endpoints for other CMS sections) | - **CMSService** – centralised CRUD for all page‑specific models <br> - **MediaService** – handle image/video upload, delete, ordering <br> - **FeaturedContentService** – manage featured images / top students | `IsAdmin` | - `AboutPageContent`, `GalleryImage`, `HomepageFeaturedImage`, `AdmissionPageContent` each store media references via `DocumentStorage` <br> - Relationships are primarily one‑to‑many (e.g., `GalleryImage` belongs to the gallery) |
| **Reports** (`/admin/reports`) | Uses same models as **Dashboard** (no extra tables) | - `GET /admin/reports/student-growth/` <br> - `GET /admin/reports/attendance/` <br> - `GET /admin/reports/exams/` (reuse exam performance) | - **ReportingService** (already used by Dashboard) | `IsAdmin` | Same as Dashboard relationships |
| **Profile** (`/admin/profile`) | - `User` (auth) <br> - `AdminProfile` (optional, can reuse `TeacherProfile`‑like model) | - `GET /admin/profile/` – current admin user info <br> - `PATCH /admin/profile/` – update password, avatar, etc. | - **ProfileService** – fetch & update admin profile | `IsAdmin` | - `AdminProfile` → `User` (one‑to‑one) |

*Note*: All new models listed in the project brief (`SubjectRequestControl`, `ClassTeacherAssignment`, `TeacherSubjectAllocation`, `FacultyAttendance`, `Exam`, `ExamSchedule`, `AnswerScriptUpload`, `EvaluationTracking`, `PublishedResult`, `Event`, `ContactSubmission`, `AdmissionApplication`, `AdmissionVerification`, `StudentRegistrationLog`, `GalleryImage`, `HomepageFeaturedImage`, `AboutPageContent`, `AdmissionPageContent`, `ExportLog`, `DocumentStorage`, `NotificationBroadcast`) are referenced where required.
